## Virtual Network Gateway

Gateway is using VMs to provide connectivity inside VPN. You can connect via VPN Tunnel.

3 methods of using Virtual Network Gateway:
- VNs-to-VNs -  connect two Virtual Networks
- Site-to-site - connect two networks
- Point-to-site - connect single device to network using VPN Tunnel

## Virtual Network Peering

Connects Virtual Networks using Azure Backbone Infrastructure.

## Express Route (Direct)

Express Route is service that allows to connect on-prem network to Azure with very high speed without using public Internet.
`Direct` version of Express Route links on-prem directly to MSEE (Microsoft Enterprise Edge) router, providing best throughput without use of 3-rd party solutions.
