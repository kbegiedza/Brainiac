# NoSQL Patterns

## Overview

## Aggregate Pattern

<Aside type="tip">

Aggregate data in entity to optimize requests.
Example: When creating e-commerce system, merge ratings and reviews with product in a single entity to optimize reads over writes.

</Aside>

## Polymorphic Pattern

<Aside type="tip">

Aggregate multiple types of objects into single collection.
Example: When creating e-commerce system, put multiple product variants into single collection leveraging no required schema.

</Aside>
